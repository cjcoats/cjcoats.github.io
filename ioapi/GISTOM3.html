Program GISTOM3


I/O API 4.0 only!!

Summary

Read multiple GIS "raster" (GRIDDED) files from one of the supported GIS formats and write the contents to a time independent GRIDDED I/O API file.

Usage 

        setenv  LLFILE    <path name for Lat-Lon file (e.g., GRID_CRO_2D)>
        setenv  FILELIST  <path name for input ASCII file-list>
        setenv  GRIDFILE  <path name for output I/O API gridded file>
For each file in FILELIST 
        setenv <file-name>  <path name>
Then 
        gistom3 [and answer the prompts]
where
${FILELIST} is list-formatted (quoted strings) with one input file per line, with 
        <file-name> <vble-name> <units> <description> <type>
Input files already on the output map-projection and grid (from LLFILE)
type is one of {BIL BIL2 BIL4 FLT, ZBIL, ZBIL2, ZBIL4, ZFLT, ASCI, ASCR, ARCI, ARCR, ARCE}
LEN( *-name ), LEN( units ) are at most 16

See Also:

  • Fortran-90 MODULE MODGISIO with routines to read and write GIS "raster" (gridded) data.

  • Program GISTOGIS to convert one GIS "raster" data-format to another.

  • Program M3TOGIS to convert GIS "raster" data to GRIDDED I/O API

  • Program GISPLOT to create GIF, JPG, or PNG tile-plots for GIS data.
